# 贝尔-拉普杜拉模型 (Bell-LaPadula Model, BLP)

**贝尔-拉普杜拉模型** (**Bell-LaPadula Model**, BLP) 是一个经典的计算机安全模型，特别用于多级安全系统（如军事系统或政府部门中的信息系统）中。它主要关注信息的**机密性** (**Confidentiality**) 而非完整性或可用性。贝尔-拉普杜拉模型的核心目标是确保信息不会被非法泄露，防止数据从高安全级别流到低安全级别。

## 背景

贝尔-拉普杜拉模型由 **David Bell** 和 **Leonard LaPadula** 于 1973 年提出，主要是为了解决多级安全系统中对信息的访问控制问题，特别是如何防止信息泄漏。 

该模型是在**强制访问控制** (**MAC, Mandatory Access Control**) 框架下工作的，它通过定义规则来限制系统中不同用户访问不同级别信息的方式。

## 核心概念

BLP 模型基于以下几个核心概念：

### 主体 (Subject)

主体是指能够访问和操作信息的实体，如用户、程序或进程。在 BLP 模型中，主体被分配一个安全级别。

### 对象 (Object)

对象是指信息或资源，如文件、数据库或其他类型的存储数据。对象同样也会被分配一个安全级别。

### 安全级别 (Security Levels)

系统中的信息和用户（主体）都被分配不同的安全级别。通常，安全级别是分层的，如：

- Top Secret (绝密)
- Secret (机密)
- Confidential (秘密)
- Unclassified (非机密，公开)

用户和信息被划分到不同的级别，而系统会确保只有符合安全要求的用户能够访问特定级别的信息。因此，BLP 模型确保机密信息不会从高安全级别流向低安全级别。

## 核心规则

BLP 模型的访问控制规则包括以下两条核心规则：

### 不上读 (简单安全规则, No Read Up, NRU)

主体不能读取高于其安全级别的信息。换句话说，用户不能访问高级别的信息。这样可以有效防止未经授权的主体接触到更高级别的信息，确保信息无法从高安全级别泄漏到低安全级别。例如，一个机密级用户不能读取绝密级的文件。

### 不下写 (* 安全规则, No Write Down, NWD)

主体不能向比自己安全级别更低的信息资源写入数据。这样则可以防止高安全级别的信息泄漏到低安全级别，从而被未授权主体访问。例如，一个绝密级用户不能将数据写入到秘密级别的文件中。

## 数学化描述

### BLP 模型描述的符号定义

- $S$: 一组主体 (Subject) 的集合
- $O$: 一组客体 (Object) 的集合
- $A$: 一组访问类型 (Access) 的集合（可读、可写、可读写、执行）
- $L$: 一组偏序安全等级集合
